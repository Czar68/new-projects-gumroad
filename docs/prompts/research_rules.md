# Research methodology (nightly improvement)

1. **Scope:** One backlog topic (or one improvement doc) per research run. 30min max.
2. **Evidence:** Every proposed change must have either (a) code diff + test, or (b) EV/math proof with +X% or reliability gain.
3. **Web:** Search "[project] [topic] best practices 2026" and "[project] EV optimization". Cite sources in the improvement doc.
4. **Verdict:** Exactly one of IMPROVEMENT (write docs/improvements, commit), FIXED (shift backlog), or NO WIN (write research_log.md).
5. **No speculation:** If no verifiable metric gain, log NoAction and do not enqueue.
